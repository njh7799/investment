# Investment Market Data

과거 시장 데이터를 일관된 형식으로 관리하고, 이를 이용해 지표 분석과 전략 백테스트를 수행하기 위한 저장소입니다.

현재는 Nasdaq Composite 지수와 QQQ·TQQQ의 일별 조정 OHLC 데이터를 수집하고 매일 자동 갱신합니다. 아래 분석 규칙은 향후 백테스트 구현에 사용할 기본 기준입니다.

## 데이터

`assets/`에는 다음 파일이 있습니다.

| 파일 | Yahoo 심볼 | 내용 |
| --- | --- | --- |
| `IXIC.csv` | `^IXIC` | Nasdaq Composite 지수 |
| `QQQ.csv` | `QQQ` | Invesco QQQ ETF |
| `TQQQ.csv` | `TQQQ` | ProShares UltraPro QQQ 및 상장 전 합성 데이터 |

CSV 스키마는 모두 `Date,Open,High,Low,Close`입니다.

- 날짜는 `YYYY.MM.DD` 형식이며 오름차순으로 정렬되고 중복되지 않습니다.
- 가격은 Yahoo Finance의 액면분할·배당 반영 조정 OHLC입니다.

### TQQQ 상장 전 합성 데이터

TQQQ 상장 전 기간은 QQQ의 일일 움직임을 3배 적용하고 최초 실제 TQQQ 가격을 기준점으로 역산합니다. 상장 이후에는 Yahoo 원본만 사용합니다.

이 합성 구간은 연구 편의를 위한 가상 시계열입니다. 운용보수, 금융비용, 배당세, 추적오차 등 실제 상품의 모든 요소를 재현하지 않으므로 실제 TQQQ 성과로 해석하면 안 됩니다.

## 설치 및 실행

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install --requirement requirements.txt
```

전체 데이터를 다시 생성하려면 다음 명령을 실행합니다.

```bash
python scripts/update_market_data.py --mode full
```

일일 갱신 모드는 Nasdaq 거래 여부와 장 마감 후 2시간 경과 여부를 확인합니다.

```bash
python scripts/update_market_data.py --mode daily
```

테스트는 다음과 같이 실행합니다.

```bash
python -m pytest --quiet
```

GitHub Actions는 매일 `23:00 UTC`에 같은 작업을 실행하며, CSV에 실제 변경이 있을 때만 커밋합니다.

## 분석 기본 규칙

### ATR(14)

ATR은 조정된 High, Low, Close로 계산합니다.

```text
True Range = max(
  High - Low,
  abs(High - Previous Close),
  abs(Low - Previous Close)
)

ATR(14) = 최근 14개 True Range의 단순이동평균
```

첫 행은 `High - Low`를 사용하고, 14개의 값이 쌓이기 전에는 ATR 기반 신호를 생성하지 않습니다. ATR은 Wilder 평활 방식이 아닌 단순이동평균입니다.

### 고점과 저점

극값은 종가 기준 ZigZag로 판정합니다. 반전 폭은 다음 두 조건을 모두 만족해야 합니다.

- 절대가격 조건: 반전 폭이 반전 확인일의 `ATR(14) × 4` 이상
- 수익률 조건: 극값 후보 종가 대비 반대 방향으로 10% 이상

상승 구간에서는 가장 높은 종가를 고점 후보로 계속 갱신합니다. 이후 종가가 고점 후보에서 위 두 조건을 모두 만족할 만큼 하락하면 그 후보를 고점으로 확정합니다. 하락 구간에서는 가장 낮은 종가를 저점 후보로 계속 갱신하고, 이후 두 조건을 모두 만족하는 상승이 발생하면 그 후보를 저점으로 확정합니다.

다음 세 날짜는 반드시 구분합니다.

- 극값 발생일: 최고 또는 최저 종가가 실제로 기록된 날
- 극값 확정일: 이후 반전이 두 조건을 만족해 앞선 후보가 극값으로 확정된 날
- 거래 체결일: 확정된 신호에 따라 주문을 실행한 날

`전고점`과 `전저점`은 가장 최근에 확정된 값만 뜻합니다. 마지막 후보는 반전 조건을 충족하기 전까지 극값으로 취급하지 않습니다.

예를 들어 고점 후보가 200이고 현재 종가가 180이면 수익률 조건은 정확히 -10%로 충족합니다. 같은 날 ATR(14)이 5라면 하락 폭 20이 `ATR × 4`와 같으므로 180의 종가가 확인되는 시점에 200을 고점으로 확정합니다.

### 백테스트 체결과 평가

- 신호는 해당 거래일 종가까지 공개된 데이터로 계산합니다.
- 주문은 신호가 확정된 다음 거래일 시가에 체결합니다.
- 데이터 마지막 날에 발생해 다음 거래일 시가가 없는 주문은 미체결로 남깁니다.
- 포트폴리오 가치는 각 거래일 종가로 평가합니다.
- 수수료, 슬리피지, 세금 및 배당 처리 방식은 전략별로 명시하며, 미지정 시 0으로 계산합니다.
- 분석 결과의 금액은 반올림하지 않고 소수점 둘째 자리까지 절삭합니다. 예를 들어 `$12,345.6789`는 `$12,345.67`로 표시합니다. 이 규칙은 원본 CSV 저장값에는 적용하지 않습니다.

고점이나 저점의 발생일에 소급해 거래한 것으로 처리하면 미래정보 편향이 생깁니다. 백테스트에서는 반드시 확정일 이후에만 해당 극값을 신호로 사용할 수 있습니다.
