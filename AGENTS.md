# Repository Instructions

이 저장소는 과거 시장 데이터 관리, 지표 분석 및 전략 백테스트를 위한 프로젝트다. 분석 정의는 `README.md`를 기준으로 하며 재현성과 미래정보 편향 방지를 우선한다.

## Market data

- `assets/IXIC.csv`, `assets/QQQ.csv`, `assets/TQQQ.csv`를 직접 편집하지 않는다. 데이터 변경은 `scripts/update_market_data.py`를 통해 재생성한다.
- CSV 스키마, 날짜 형식과 기존 데이터 검증 조건을 유지한다.
- 액면분할과 배당이 반영된 조정 OHLC를 사용하고 원본 가격을 절삭하지 않는다.
- TQQQ 최초 실제 데이터 이전만 QQQ 기반 3배 합성을 허용한다. 실제 데이터가 시작된 이후의 누락일을 합성해서 채우지 말고 오류로 처리한다.
- 전체 재생성, 최신 거래일 검증, 검증 완료 후 파일 교체, 변경이 있을 때만 커밋하는 동작을 보존한다.

## Analysis conventions

- ATR, ZigZag, 체결 및 평가 방식은 `README.md`의 기본 규칙을 따른다. 다른 규칙을 쓰는 전략은 그 차이를 코드와 결과에 명시한다.
- 극값 발생일, 확정일, 거래 체결일을 구분하고 확정되지 않은 후보를 전고점이나 전저점으로 사용하지 않는다.
- 신호 계산에는 당시 공개된 데이터만 사용한다. 극값 발생일로 거래를 소급하는 등 미래정보 편향을 허용하지 않는다.
- 내부 계산과 원본 데이터의 정밀도를 유지하고, 금액 절삭은 결과 표시에만 적용한다.
- 거래비용과 배당 처리 가정을 백테스트 결과에 명시한다.

## Verification

- 변경 후 `python -m pytest --quiet`를 실행한다.
- 데이터 로직에는 스키마, 날짜, 결측치 및 OHLC 불변식 검증을 유지한다.
- 분석 로직에는 ATR 준비 기간과 임계값 경계, 후보 갱신과 확정, 마지막 미확정 후보, 다음 거래일 체결 테스트를 포함한다.
